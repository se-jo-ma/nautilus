## Learnings
- Full `nautilus` package import chain requires `fathom` + `httpx` etc.; isolated module loading needed for verify in dev environment
- InfluxDB adapter follows same pattern as postgres/elasticsearch: ClassVar source_type, connect/execute/close, scope-to-query mapping
- Flux queries use `|> filter(fn: (r) => ...)` for tag/field filters and `|> range(start, stop)` for time constraints
- `_time` scope constraints are lifted into the Flux `range()` call rather than filter predicates
