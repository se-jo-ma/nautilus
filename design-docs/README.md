# Nautilus Design Docs

The Nautilus design is split across five documents. Read them in order, or jump to the one you need.

1. [01-overview.md](./01-overview.md) — Thesis, problem statement, version scope, how a request flows through the system, and intent analysis.
2. [02-core-engine.md](./02-core-engine.md) — Cumulative exposure tracking, the Fathom DSL, the routing/policy engine, and result synthesis.
3. [03-living-brain.md](./03-living-brain.md) — v2 knowledge evolution: meta-rules, LLM-as-knowledge-engineer, similarity scoring, rule conflict detection, relationships, and proactive suggestions.
4. [04-architecture-operations.md](./04-architecture-operations.md) — Persistence, concurrency, failure modes, attestation & audit, adapter security, source registry, classification, agent interface, adapters, deployment, scaling, observability.
5. [05-ecosystem-roadmap.md](./05-ecosystem-roadmap.md) — Relationship to Fathom/Bosun, RETE, testing strategy, out-of-scope list, known weaknesses & open questions, development roadmap, open source strategy, and the Expert Networks future direction.

The monolithic source document `../design-v2.md` is retained until the split is reviewed.
